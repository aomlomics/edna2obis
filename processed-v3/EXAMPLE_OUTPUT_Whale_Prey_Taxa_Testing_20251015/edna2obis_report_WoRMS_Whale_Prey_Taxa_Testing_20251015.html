Detailed Taxa Assignment Information

The table below (taxa_assignment_INFO.xlsx, sheet taxa_assignment_INFO) provides a comprehensive look at the results of the taxonomic matching process. It includes all potential matches found for each unique verbatimIdentification from your raw data, not just the single best match chosen for the final occurrence file. This allows for manual review and complete transparency.

  • verbatimIdentification: The original, unaltered taxonomic string from your input data.
  • cleanedTaxonomy: A standardized version of the verbatim string used for matching.
  • selected_match: (WoRMS and GBIF) Primary column to use when reading this file: True on the one row per verbatimIdentification that was chosen for the final occurrence file (or, for taxassign-only runs, the row that would be chosen in the full pipeline). Other rows are alternate candidates for review.
  • scientificName: The scientific name of the match returned by the taxonomic service (WoRMS or GBIF).
  • confidence: A score from 0-100 indicating GBIF's confidence in the match (GBIF only).
  • ambiguous: (WoRMS only) A flag indicating if multiple potential matches were found for the verbatim string.
  • replaced_unaccepted: (WoRMS only) True when WoRMS resolved an unaccepted name to its accepted valid name.
  • unaccepted_match: (WoRMS only) True when this row is the unaccepted (queried) taxon shown so you can compare its assignment score to the accepted name. Whether it can be selected for the occurrence core is controlled by worms_consider_unaccepted_for_selection in config.
  • higherClassification: (WoRMS only, optional) Pipe-separated higher-taxon lineage from WoRMS, used to preserve intermediate ranks without creating many sparse columns.
  • ranks_matched: (WoRMS only) Count of verbatim lineage tokens found in the matched classification.
  • ranks_provided: (WoRMS only) Total number of verbatim lineage tokens considered for scoring.
  • assignment_score: (WoRMS only) Ratio = ranks_matched / ranks_provided.
  • environment: (WoRMS only) Semicolon-separated WoRMS habitat labels for the selected match, such as marine or freshwater. Blank means WoRMS did not provide habitat data.
  • consistency_check: (GBIF only) A check to ensure the kingdom of the match is consistent with the kingdom in the verbatim string. Helps identify homonym errors.
